## Рекомендуемый порядок исправлений

1. **Сначала критические баги и навигация (Фаза 1 + 7.1 + 7.2a)**
   - Исправить неправильные колонки статистики, callback для вратарей, шаблон времени гола.
   - Вернуть `reply_markup`, чтобы бот снова стал полностью проходимым в UI.

2. **Затем согласование архитектуры и схемы (1.2c/5.1 + 5.5)**
   - Объединить конфиг в один модуль.
   - Закрыть mismatch по полям схемы/кода (`is_shootout`/`is_shootouts` и т.д.).

3. **Далее качество кода (Фаза 3)**
   - Убрать star-импорты, заменить `print()` на `logging`, добавить аннотации и убрать закомментированный код.
   - После этого безопаснее делать следующие изменения и ревью.

4. **Потом инфраструктура и тестирование (Фаза 2 + Фаза 6)**
   - Поднять Docker/Compose, затем CI.
   - Добавить тесты и docstrings, чтобы стабилизировать дальнейшие UX/feature-изменения.

5. **В конце продуктовые улучшения (Фаза 7 остальное + 5.4)**
   - Расширять UX/контент и добавить кэширование под часто запрашиваемые сценарии.
   - Это даст максимум пользы пользователю без риска сломать базовый поток.
